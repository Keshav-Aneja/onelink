The Problem
What we have
  • Hierarchical collections (folders)
  • Full-text search via pg_trgm
  • Rich metadata already scraped: keywords, author, site_description, OG fields
  • Star / RSS filters on LinksContent
What's missing
  • Scraped keywords field is stored but never surfaced to the user
  • No way to filter across collections ("show me all my Python links")
  • Nested folders are great for deliberate org, terrible for serendipitous retrieval
  • No cross-collection discovery axis exists
2
Approach: Deterministic Tag Derivation

No LLM. No third-party API. The scraper already extracts everything we need. The algorithm is a pure string-processing pipeline that runs synchronously on the already-scraped metadata at link-save time.

Tag Derivation Sources (priority order)
# Source field Already in DB? How we use it Example output
1 keywords Yes Split on , or ;, trim, lowercase, dedupe react, typescript, hooks
2 articleTag (OG / article meta) Scraped, not stored Promote to tag candidates; store in new tags table tutorial, open-source
3 articleSection Scraped, not stored Single-word section becomes a tag engineering, design
4 author / dcCreator Yes Optional: author name → tag (user-opt-in only) @dan-abramov
5 Domain name from URL Derived at runtime Hostname stripped to label: youtube.com → youtube youtube, github
6 ogType Scraped, not stored Map: video.other → video, article → article video, article
Normalization rules
  • Lowercase all candidates
  • Strip leading/trailing whitespace and punctuation
  • Replace spaces with hyphens (machine learning → machine-learning)
  • Deduplicate before inserting
  • Max tag length: 40 chars (truncate with ellipsis)
  • Ignore stop-words list: the, a, an, in, of, and, or, to, for, with, is, are, be…
  • Ignore single-character candidates
  • Cap auto-suggestions at 8 tags per link

// TABLE: tags
await knex.schema.createTable('tags', (t) => {
  t.uuid('id').primary().defaultTo(
    knex.raw('gen_random_uuid()')
  );
  t.string('name', 40).notNullable();
  t.uuid('owner_id')
    .notNullable()
    .references('id')
    .inTable('users')
    .onDelete('CASCADE');
  t.boolean('is_auto').defaultTo(true);
  t.timestamps(true, true);
  t.unique(['name', 'owner_id']);
  t.index(['owner_id']);
});
// TABLE: link_tags (junction)
await knex.schema.createTable('link_tags', (t) => {
  t.uuid('link_id')
    .notNullable()
    .references('id')
    .inTable('links')
    .onDelete('CASCADE');
  t.uuid('tag_id')
    .notNullable()
    .references('id')
    .inTable('tags')
    .onDelete('CASCADE');
  t.boolean('confirmed')
    .defaultTo(false);
  t.primary(['link_id', 'tag_id']);
  t.index(['tag_id']);
  t.index(['link_id']);
});
Key design decisions
  • Tags are per-user (scoped by owner_id) — no global tag namespace, no cross-user leakage
  • confirmed flag tracks whether the user explicitly accepted or added a tag vs. a pending auto-suggestion
  • is_auto on the tag row distinguishes user-created tags from auto-derived ones for future analytics
  • Unique constraint on (name, owner_id) means the same tag string is reused across links — no duplication
  • All deletes cascade: deleting a link removes its link_tags; deleting a user removes all tags and link_tags

Tag Deriver — pseudocode
// packages/scraper/src/tag-deriver.ts

const STOP_WORDS = new Set(['the', 'a', 'an', 'in', 'of', 'and', 'or', 'to', 'for', ...]);
const OG_TYPE_MAP: Record<string, string> = {
  'video.other': 'video', 'article': 'article', 'music.song': 'music'
};

export function deriveTags(meta: WebsiteMetadata, url: string): string[] {
  const candidates: string[] = [];

  // 1. keywords field (already stored)
  if (meta.keywords) {
    candidates.push(...meta.keywords.split(/[,;]/));
  }

  // 2. article tags (scraped but not stored)
  if (meta.articleTag) candidates.push(meta.articleTag);
  if (meta.articleSection) candidates.push(meta.articleSection);

  // 3. domain label
  try {
    const host = new URL(url).hostname.replace(/^www\./, '').split('.')[0];
    candidates.push(host);
  } catch {}

  // 4. OG type normalisation
  if (meta.ogType && OG_TYPE_MAP[meta.ogType]) {
    candidates.push(OG_TYPE_MAP[meta.ogType]);
  }

  // normalise
  return [...new Set(
    candidates
      .map(s => s.trim().toLowerCase().replace(/\s+/g, '-').replace(/[^\w-]/g, ''))
      .filter(s => s.length > 1 && s.length <= 40 && !STOP_WORDS.has(s))
  )].slice(0, 8);
}

Get links with their tags (JOIN)
// links.repository.ts — updated getLinksForCollection()
knex('links')
  .select([
    'links.*',
    knex.raw('COALESCE(json_agg(json_build_object('
      + "'id', t.id, 'name', t.name, 'confirmed', lt.confirmed"
      + ") FILTER (WHERE t.id IS NOT NULL), '[]') AS tags")
  ])
  .leftJoin('link_tags as lt', 'lt.link_id', 'links.id')
  .leftJoin('tags as t', 't.id', 'lt.tag_id')
  .where({ 'links.owner_id': ownerId, 'links.parent_id': collectionId })
  .groupBy('links.id');
Filter links by tag (AND semantics)
// GET /links?tag=react&tag=typescript
knex('links')
  .join('link_tags as lt', 'lt.link_id', 'links.id')
  .join('tags as t', 't.id', 'lt.tag_id')
  .where('links.owner_id', ownerId)
  .whereIn('t.name', tagFilters)          // ['react', 'typescript']
  .groupBy('links.id')
  .havingRaw('COUNT(DISTINCT t.name) = ?', [tagFilters.length]);
Upsert tag + link_tag (at save time)
async function upsertTagsForLink(
  linkId: string, ownerId: string, tagNames: string[]
): Promise<void> {
  for (const name of tagNames) {
    // insert tag if not exists, return id
    const [tag] = await knex('tags')
      .insert({ name, owner_id: ownerId, is_auto: true })
      .onConflict(['name', 'owner_id'])
      .merge()           // idempotent
      .returning('id');

    // attach to link as unconfirmed suggestion
    await knex('link_tags')
      .insert({ link_id: linkId, tag_id: tag.id, confirmed: false })
      .onConflict(['link_id', 'tag_id'])
      .ignore();
  }
}

Non-Goals & Constraints
Explicitly out of scope
  • No LLM or AI API calls — zero extra cost
  • No NLP library (spaCy, compromise.js) — pure string ops
  • No global/shared tag namespace — tags are user-scoped only
  • No tag hierarchy / parent-child tags
  • No tag-based access control or sharing
  • No bulk re-tagging UI (backfill is a one-time script)
Edge cases handled
  • Links with no keywords/OG — get only the domain tag
  • Duplicate tags across links — idempotent upsert handles it
  • Very long keyword strings — truncated at 40 chars
  • Non-ASCII tag candidates — replace(/[^\w-]/g, '') strips them
  • User deletes a tag — CASCADE removes all link_tags references
  • Retroactive backfill — uses existing keywords column, no re-scrape needed
